import { PackageManagers } from 'starlight-package-managers'

`starlight-package-managers` exports an Astro component that you can use in any MDX files of your Starlight documentation site.

<Preview>

```mdx title="src/content/docs/example.mdx"
---
title: My Docs
---

import { PackageManagers } from 'starlight-package-managers'

<PackageManagers pkg="astro" />
```

<PackageManagers slot="preview" pkg="astro" />

</Preview>

## Command Types

The component provided by `starlight-package-managers` supports various types of commands that can be specified using the `type` prop and defaults to the [`add` type](#add) if none is provided.

### `add`

The `pkg` prop is used to specify the name(s) of the package(s) to install. You can specify one or more package names by separating them with a space.

<Preview>

```mdx title="src/content/docs/example.mdx" 'pkg="astro @astrojs/starlight"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers pkg="astro @astrojs/starlight" />
```

<PackageManagers slot="preview" pkg="astro @astrojs/starlight" />

</Preview>

#### Development dependencies

You can use the `dev` prop to specify that the package(s) should be installed as a development dependency.

<Preview>

```mdx title="src/content/docs/example.mdx" "dev"
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers pkg="astro" dev />
```

<PackageManagers slot="preview" pkg="astro" dev />

</Preview>

### `create`

To setup new or existing project, you can use the `create` type.

<Preview>

```mdx title="src/content/docs/example.mdx" 'type="create"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="create" pkg="astro@latest" />
```

<PackageManagers slot="preview" type="create" pkg="astro@latest" />

</Preview>

### `dlx`

To fetch and execute a package binary, without installing it as a dependency, you can use the `dlx` type and specify extra arguments using the `args` prop.

<Preview>

```mdx title="src/content/docs/example.mdx" 'type="dlx"'
<PackageManagers type="dlx" pkg="serve" args="public" />
```

<PackageManagers slot="preview" type="dlx" pkg="serve" args="public" />

</Preview>

### `exec`

To execute a package binary, you can use the `exec` type and specify extra arguments using the `args` prop.

<Preview>

```mdx title="src/content/docs/example.mdx" 'type="exec"' 'args="add solid"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="exec" pkg="astro" args="add solid" />
```

<PackageManagers slot="preview" type="exec" pkg="astro" args="add solid" />

</Preview>

### `install`

To install all dependencies, you can use the `install` type, with no other props specified.

<Preview>

```mdx title="src/content/docs/example.mdx" 'type="install"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="install" />
```

<PackageManagers slot="preview" type="install" />

</Preview>

### `remove`

The `pkg` prop is used to specify the name of the package to remove.

<Preview>

```mdx title="src/content/docs/example.mdx" 'type="remove"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="remove" pkg="@astrojs/starlight" />
```

<PackageManagers slot="preview" type="remove" pkg="@astrojs/starlight" />

</Preview>

### `run`

To run a script defined in the package's manifest file, you can use the `run` type and specify the name of the name of the script using the `args` prop.

<Preview>

```mdx title="src/content/docs/example.mdx" 'type="run"' 'args="dev"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="run" args="dev" />
```

<PackageManagers slot="preview" type="run" args="dev" />

</Preview>

## Extra arguments

You can provide extra arguments to any command using the `args` prop.

<Preview>

```mdx title="src/content/docs/example.mdx" 'args="--template starlight"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="create" pkg="astro@latest" args="--template starlight" />
```

<PackageManagers
  slot="preview"
  type="create"
  pkg="astro@latest"
  args="--template starlight"
/>

</Preview>

## Comment

You can include a comment before a command using the `comment` prop.

<Preview>

```mdx title="src/content/docs/example.mdx" 'comment="create a new project"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="create" pkg="astro" comment="create a new project" />
```

<PackageManagers
  slot="preview"
  type="create"
  pkg="astro"
  comment="create a new project"
/>

</Preview>

You can also reference the name of the selected package manager in a comment using the `{PKG}` placeholder.

<Preview>

```mdx title="src/content/docs/example.mdx" "{PKG}"
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers
  type="create"
  pkg="astro"
  comment="create a new project with {PKG}"
/>
```

<PackageManagers
  slot="preview"
  type="create"
  pkg="astro"
  comment="create a new project with {PKG}"
/>

</Preview>

## Prefix

You can include a prefix before a command, for example to define an environment variable, using the `prefix` prop.

<Preview>

```mdx title="src/content/docs/example.mdx" 'prefix="DEBUG=true"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers type="run" args="dev" prefix="DEBUG=true" />
```

<PackageManagers slot="preview" type="run" args="dev" prefix="DEBUG=true" />

</Preview>

## Appearance

By default, commands are rendered with a terminal frame around them and an icon indicating the package manager.
You can disable this behavior and optionally provide a title for the frame.

### Frame

To disable the terminal frame surrounding commands when using [Expressive Code](https://starlight.astro.build/reference/configuration/#expressivecode), you can use the `frame` prop and set it to `none`.

<Preview>

```mdx title="src/content/docs/example.mdx" 'frame="none"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers pkg="astro" frame="none" />
```

<PackageManagers slot="preview" pkg="astro" frame="none" />

</Preview>

### Icons

By default, `starlight-package-managers` displays a package manager icon next to its name.
You can disable this behavior using the `icons` prop and set it to `false`.

<Preview>

```mdx title="src/content/docs/example.mdx" 'icons={false}'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers pkg="astro" icons={false} />
```

<PackageManagers slot="preview" pkg="astro" icons={false} />

</Preview>

### Title

When using [Expressive Code](https://starlight.astro.build/reference/configuration/#expressivecode) and the default terminal frame, you can provide a title using the `title` prop.

<Preview>

```mdx title="src/content/docs/example.mdx" 'title="Install dependencies"'
import { PackageManagers } from 'starlight-package-managers'

<PackageManagers pkg="astro" title="Install dependencies" />
```

<PackageManagers slot="preview" pkg="astro" title="Install dependencies" />

</Preview>
